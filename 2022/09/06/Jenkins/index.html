¶使用

在主页新建一个空项目，创建一个新的SpringBoot项目，将项目托管到gitlab。编写简单的代码，将代码提交到master分支。随后在gitlab中提交合并请求，将master代码合并到main分支中。

Jenkins

Jenkins是一款开源，基于Java所搭建的持续集成持续部署CI/CD的平台

¶安装

Jenkins本质就是一个Java项目，在官网下载后，使用java -jar命令启动即可，访问8080端口，安装推荐插件。

建议不要将Jenkins和gitlab安装在同一台机器上，Jenkins和gitlab都会占用8080端口。

¶使用

新建一个任务，但是没有Maven构建，需要额外安装插件

在系统管理->插件管理中安装相关Maven集成的插件。安装完成之后，新建任务时可以看见有一个猫头鹰的选项，就是一个Maven任务。下一步有许多的配置任务要填写。

  1. 源码管理：

    选择Git，填写仓库地址，Jenkins会自动帮我们测试这个地址是否有效，这需要Jenkins服务器安装有Git

    如果项目有凭证则添加凭证

  2. 修改分支，由于代码都合并到main分支，默认是master分支

  3. 配置maven，在全局配置栏里，新增Maven

  4. 配置完成之后可以在主面板点击运行，开始自动构建

  5. 连接测试服务器，需要安装一个插件public over ssh，安装完成之后，重新配置一下项目，在Post Steps中勾选Send files or execute commands over SSH

    Name中默认为空，因为还没有配置测试服务器，在系统管理-系统配置中添加服务器

    然后在项目中配置测试服务器

  6. 让Jenkins自动运行Java程序，一般我们会使用命令java -jar，但是这样会阻塞控制台，所以通常使用nohup java -jar xxx.jar &。但是这个命令也是存在问题的，如果在真机下输入这条命令，控制台还是会阻塞， 直到用户输入一个回车才会恢复正常，Jenkins会认为这个命令没有结束。同时Jenkins不会停止上一次测试启动的Jar包。

    但是还是没有处理停止前一个Jar包，这些前置清理工作，我们需要写一个Shell脚本来完成

¶Gitlab自动构建

在生产环境中我们希望提交代码后自动构建，这就需要用到Jenkins的构建触发器。

随后访问JENKINS_URL/job/first/build?token=123456就可以将构建任务提交到Jenkins里。但是这个前提是我们已经登录到了Jenkins，如果在其他浏览器，或者第三方HTTP平台，会提示没有权限。这时候需要安装一个插件Build Authorization Token Root能够支持我们匿名提交构建任务。

安装完成之后，访问JENKINS_URL/buildByToken/build?job=first&token=123456

我们希望的是在提交代码之后触发构建任务，所以我们要把这个地址交给Gitlab

在第一次填入地址时，会提示本地地址不被允许，需要更改Gitlab的一些设置。在菜单->管理员->设置->网络->出站请求->勾选允许来自 web hooks 和服务对本地网络的请求。

这种方式在开发中不是很常用，它会提交多次构建请求，严重消耗系统资源。使用手动构建或者定时构建更多。

几种构建方式：

  1. 快照依赖构建：当依赖的快照被构建时执行本job
  2. 触发远程构建：远程调用本job的restapi时执行job
  3. job依赖构建：当依赖的job被构建时执行本job
  4. 定时构建
  5. 向GitHub提交代码时触发
  6. 定期检查代码变更Poll SCM

¶定时构建

Jenkins的cron表达式不是标准的cron表达式，普通的cron表达式用五个星号表示，Jenkins多了一个H表示哈希散列，目的是如果在同一个时间有多个任务构建，可以避免并发任务冲突的问题。H(1-30)表示从1到30中取一个随机值。

流水线

流水线既能作为任务的本身，也能作为Jenkinsfile。使用流水线可以让任务从ui手动操作转变为代码操作，从shell命令到配置文件，更适合大型项目，可以让团队其他开发者同时参与进来，也可以构建ui不能完成的复杂构建逻辑。
