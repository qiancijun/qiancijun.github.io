提高整体性能

数据库经常被多个客户访问，对处理什么请求以及用什么次序处理进行管理是MySQL的任务。INSERT操作可能很耗时（特别是有很多索引需要更新时），而且它可能降低等待处理的SELECT语句的性能。
如果数据检索是最重要的（通常是这样），则可以通过在INSERT和INTO之间添加关键字LOW_PRIORITY，指示MySQL降低INSERT语句的优先级。如INSERT LOW_PRIORITY INTO。这也适用于UPDATE和DELETE子句。

IGNORE关键字

如果UPDATE语句更新多行，并且在更新这些行中的一行或多行时出现一个错误，则整个UPDATE操作被取消（错误发生前更新的所有行被恢复到它们原来的值）。即使是发生错误，也继续进行更新，可使用IGNORE关键字，UPDATE IGNORE customers。

数值型

数据类型 说明
BIT 位字段，1~64位。（在MySQL 5之前，BIT在功能上等价于TINYINT）
BIGINT 整数值，支持-9223372036854775808~9223372036854775807。无符号型UNSIGNED，范围很大，不用记住
BOOLEAN或BOOL 布尔标志，或者为0或者为1
DECIMAL或DEC(M,D) 精度可变的浮点值，默认精度(10, 0)
DOUBLE(M,D) 双精度浮点值
FLOAT(M,D) 单精度浮点值
INT或INTEGER 整数值。支持$\textit{-}2^{31}$ ~ $2^{31} \textit{-} 1$
MEDIUMINT 整数值。支持$\textit{-}2^{23}$ ~ $2^{23} \textit{-} 1$
REAL 4字节的浮点值
SMALLINT 整数值。支持$\textit{-}2^{15}$ ~ $2^{15} \textit{-} 1$

注：存储货币数据类型：
MySQL中没有专门存储货币的数据类型，一般情况下使用DECIMAL(8,2)
M:有效数字的个数，整数加小数的位数 D:小数点后保留的位数

  1. 创建一个无符号的数据类型

    注：如果不设置有符号或无符号，默认是有符号，如果插入的值超出数据类型的范围，那么会默认插入临界值。如果不设置长度会有默认的长度。长度代表显示结果的长度，长度不够用0填充且必须搭配关键字zerofill，同时int变为无符号

串数据

数据类型 说明
CHAR 1~255个字符的的定长串。它的长度必须在创建时指定
ENUM 接受最多64K个串组成的一个预定义集合的某个串
TEXT 最大长度为64K的边长文本
LONGTEXT 与TEXT相同，但最大长度为4GB
MEDIUMTEXT 与TEXT相同，但最大长度为16K
SET 接受最多64个串组成的一个预定义集合的零个或多个串
VARCHAR 长度可变，最多不超过255字节

char和varchar

char varchar
固定长度的字符串 可变长度的字符串
比较耗费 比较节省
效率高 效率低

日期和时间类型

数据类型 说明
DATE 表示1000-01-01~9999-12-31，格式为YYYY-MM-DD
TIME 格式为HH:MM:DD
DATETIME DATE与TIME的组合
TIMESTAMP 时间戳
YEAR 用两位数字表示，70(1970)~69(2069)

数据类型 说明
BLOB Blob最大长度为64KB
MEDIUMBLOB MEDIUMBlob最大长度为16MB
LONGBLOB LONGBlob最大长度为4GB
TINYBLOB TINYBlob最大长度为255字节
