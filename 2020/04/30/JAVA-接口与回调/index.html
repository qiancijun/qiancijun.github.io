接口

¶接口概念

在Java程序设计语言中，$\color{red}{接口不是类}$，而是对类的一组特殊需求，这些类要遵从接口描述的统一格式进行定义。比如Arrays类中的sort方法可以对对象数组进行排序，但前提是该类必须实现了Comparable接口。

在实现Comparable<Employee>接口的类中，必须提供compareTo方法。

接口中的所有方法自动地属于public。因此在接口中声明方法时，不必提供关键字public。

¶实现接口

为了让一个类实现一个接口，通常需要下面两个步骤：

  • 将类声明为实现给定的接口。
  • 对接口中$\color{red}{所有}$方法进行定义。
  • 要将类声明为实现某个接口，需要使用关键字：implements:

class Employee implements Comparable

¶接口的特性

接口不是类，尤其不能使用一个new运算符实例化一个接口：

x = new Comparable(); //错误

但是可以声明接口的变量：

Comparable x; //正确

接口变量必须引用实现了接口类的对象：

x = new Employee();

可以使用instanceof检查一个对象是否属于某个特定类，也可以使用instance检查一个对象是否实现了某个特定的接口：

if (anObject instanceof Comparable){}

尽管每个类只能够拥有一个超类（在java中只有单继承，在c++中有多继承），但却可以实现多个接口，实现多个接口时接口名除用逗号隔开：

class Employee implements Cloneable, Comparable

¶接口与抽象类

既然已经有了抽象类，为什么还要接口？原因就在于Java不同于C能实现多继承，在Java中每个类只能扩展一个类。而接口可以实现在C中的多重继承，同时能够避免多重继承的复杂性和低效性。

¶接口中的方法

接口允许拥有自己的私有方法，默认方法和静态方法。其中静态方法只能按照接口名.方法名来进行调用。接口中允许拥有变量，但是该变量为静态常量public static final，修饰符可省略。接口与接口直接可以多继承。

回调

回调是一种常见的设计模式。（书上的回调示例以及解释个人觉得讲的不是很清楚，以下都是我的理解）。加入现在我有两个类，一个叫A，一个叫B。其中A叫上级，B叫下级。回调机制一般都出现在分级类的方法中。在上级类A中有一个方法叫a()，在下级类B中有一个方法b()，而b()里又调用了方法a()，那么a()就被称为回调函数。这个例子可能还是很难懂，以领导和员工为例，领导要求员工干活，员工干完了回馈给领导信息“我做完了”。在这种模式中，可以指出某个特定事件发生时应该采取的动作。
