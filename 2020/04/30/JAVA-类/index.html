内部类

在Java中允许一个类的定义位于另外一个类的内部，前者称为内部类，后者称为外部类。

¶特点

  • 内部类方法可以访问该类定义所在的作用域中的数据，包括私有的数据。
  • 内部类可以对同一个包中的其他类隐藏起来。
  • 当想要定义一个回调函数但不想编写大量代码时，使用$\color{blue}{匿名}$内部类比较便捷。

¶内部类

用一个例子来说明如何定义内部类，以及内部类的特点：

  • 可以定义多个内部类
  • 可以声明为final、private、protected、static
  • 一旦内部类声明为static，此时就不能使用外层类非static的成员变量。

¶局部内部类

在上面这个例子中，仅在setInfo的方法中调用了内部类。那么我们可以在setInfo中定义内部类，在方法中定义类被称为局部内部类。

¶匿名内部类

假如只创建这个类的一个对象，就不必命名了。

¶静态内部类

有时候，使用内部类只是为了把一个类隐藏在另外一个类的内部，并不需要内部类引用外围类对象。为此可以将内部类声明为static，以便取消产生的引用。

下面是一个使用静态内部类的典型例子。计算一个数组中最小值和最大值的问题。编写两个方法，一个方法用于计算最小值，另一个方法用于计算最大值。在调用这两个方法的时候，数组被遍历两次。如果只遍历数组一次，并能够同时计算出最小值和最大值，这样就可以提高效率了。

  • 注：
    • 只有内部类可以声明为static。静态内部类的对象除了没有对生成它的外围类对象的引用特权外，与其他所有内部类完全一样。
    • 在内部类不需要访问外围类对象的时候，应该使用静态内部类。
    • 与常规内部类不同，静态内部类可以有静态域和方法。
    • 声明在接口中的内部类自动称为static和public类。

静态类

$\color{red}{如果一个类要被声明为static，只有一种情况，就是静态内部类。}$

¶特点

  1. 全局唯一，任何一次修改都是全局性的影响
  2. 只加载一次，优先于非静态类
  3. 使用方法上不依赖于实例对象
  4. 生命周期属于类级别，从JVM加载开始到JVM卸载结束
  5. 静态内部类和静态方法一样，只能访问静态的成员变量和方法，不能访问非静态的方法和属性。
  6. 静态内部类可以声明普通成员变量和方法，而普通内部类不能声明static成员变量和方法。
  7. 静态内部类可以单独初始化

Inner i = new Outer.Inner();

  • 注意要点：
    • 非静态方法既可以访问静态数据成员 又可以访问非静态数据成员，而静态方法只能访问静态数据成员；
    • 非静态方法既可以访问静态数据成员 又可以访问非静态数据成员，而静态方法只能访问静态数据成员；
    • 非静态方法既可以访问静态数据成员 又可以访问非静态数据成员，而静态方法只能访问静态数据成员；

原因：

  1. 因为静态方法和静态数据成员会随着类的定义而被分配和装载入内存中，而非静态方法和非静态数据成员只有在类的对象创建时在对象的内存中才有这个方法的代码段。
  2. static修饰的静态方法会随着类的定义而被分配和装载入内存中，编译器只为整个类创建了一个静态变量的副本，也就是只分配一个内存空间，虽然可能有多个实例，但这些实例共享该内存，特别值得注意的是，任何一个对象对静态数据成员的修改，都会影响其它对象。
  3. 静态不能引用非静态这一特性，是由于静态的会随着类的定义而被分配和装载入内存中这一关键点决定的；如果静态引用了非静态的，根本无法从内存中找到非静态的代码段，势必会出错，这种做法是Java虚拟机决不允许的。

¶静态域

如果将域定义为static，每个类中只有一个这样的域。而每一个对象对于所有的实例域却都有一份自己的拷贝。

¶静态方法

静态方法是一种不能向对象实施操作的方法。例如Math类下的方法。可以认为静态方法是没有this参数的方法。

¶使用

什么时候会用到static这个关键字呢？

  1. 需要在静态方法中调用的变量，一定要设为静态。
  2. 被static修饰的方法或变量会优先于类实度例的产生而常驻于内存。也就是说有可能类的实例还没用用到的时候，该类里的静态函知数已经在内存中出现了。
  3. 被static修饰的成员变量和成员方法独立于该类的任何对象。也就是说，它不依赖类特定的实例，被类的所有实例共享。
