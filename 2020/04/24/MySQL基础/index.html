MySQL数据库基础

DB与DBMS

数据库(Database)：保存有组织的数据的容器

数据库软件(Database Management System)：数据库管理系统

数据库结构

表

表是一种$\color{red}{结构化}$的文件，可用来存储某种特定类型的数据。数据库中的每个表都有一个名字，用来标识自己。此名字是$\color{red}{唯一}$的。

模式

关于数据库和表的布局及特性的信息

列

表中的一个字段，所有表都是由一个或多个列组成的。

行

表中的数据是按行存储的，所保存的每个记录存储在自己的行内。

主键

一列(或一组列)，其值能唯一区分表中每个行。

Mysql命令

命令格式

1.命令输入在mysql>之后；
2.命令用;或\g结束，仅按Enter不执行命令；
3.输入help获得帮助
4.输入quit或exit退出命令行程序

连接

连接本地数据库

mysql -u root -p
-h 主机名， 表示要连接的数据库的主机或者IP
-u 用户名，表示连接数据库的用户名
-P 端口，表示要连接数据库的端口，默认是3306，可以不写。
-p 表示要连接数据库的密码

连接远程数据库

mysql -h ip地址 -u 用户名 -p
例如 mysql -h 10.10.25.159 -u root -p

退出数据库

quit; 或 exit;

数据库和表

数据库、表、列、用户等信息被存储在数据库和表中。但是内部表一般不直接访问。可用show命令来显示这些信息。

作用：查看当前库的所有表
分析：返回当前选择的数据库内可用表的列表

mysql> USE 库名
作用：打开指定的库

);
作用：创建表

mysql> desc 表名;
作用：查看表结构(describe的缩写)

DQL语言

基础查询

语法：
mysql> select 查询列表 from 表名
类似于与JAVA中的System.out.println();
查询列表可以是：表中的字段，常量值、表达式、函数。
查询结果是一个虚拟的表格

  1. $\color{blue}{检索单个列}$

    mysql> SELECT prodname FROM praducts;

  2. $\color{blue}{检索多个列}$

    mysql> SELECT prod_id, prod_name, prod_price products;
    中间用$\color{red}{逗号}$隔开

  3. $\color{blue}{使用通配符}$

    mysql> SELECT * FROM products;
    如果给定一个通配符(*)，则返回表中的所有列。

  4. $\color{blue}{使用通配符}$
    一般，除非确实需要表中的每个列，否则最好别使用*通配符。虽然使用通配符可能会减少操作，不用明确列出所有列，但检索不需要的列通常会降低检索和应用程序的性能。

  5. 如果字段名除和关键字重复，可以利用着重号“ ` ”来引用字段。

  6. $\color{blue}{查询常量值}$

    mysql> SELECT 100;
    mysql> SELECT ‘Qianci_Jun’;

  7. $\color{blue}{查询表达式}$

    mysql> SELECT 100*100;

  8. $\color{blue}{查询函数}$

    mysql> SELECT version(); #调用该方法，查询它的返回值

  9. $\color{blue}{字段别名}$

    mysql> SELECT 100 * 100 $\color{red}{AS}$ 一万;
    多个字段起别名：
    mysql> SELECT 100 * 100 AS 一万, person_name AS 名字 FROM 表名;

    • 注：
      • 便于理解
      • 如果要查询的字段有重名的情况，使用别名可以区分开来
      • 其中AS可以省略不写
      • 如果别名中有特殊符号（例如：# 空格），别名用双引号包括起来

  10. $\color{blue}{去重}$

    关键字：distinct
    mysql> SELECT DISTINCT 字段 FROM 表名;

  11. $\color{blue}{+号的用法}$

    仅起算术运算的作用，如果有一个字段为字符型，则会自动转换成整数型。

  12. $\color{blue}{concat(str1, str2…)函数}$

    比如：查询员工的姓和名并且连接成一个字段，显示为姓名
    mysql> SELECT CONCAT(last_name, first_name) AS 姓名 FROM employees;

  13. $\color{blue}{ifnull(exp1, exp2)函数}$

    作用：判断是否为空，第一个参数代表可能为null的表达式，第二个参数代表，如果第一个参数为null为真那么返回exp2对应的值。
    并且可以在concat函数中调用ifnull。

限制结果

SELECT语句返回所有匹配的行，它们可能是指定表中的每个行。为了返回第一行或前几行，可以使用LIMIT子句。

SELECT
        prod_name
FROM
        products
LIMIT
        5;

分析：此语句使用SELECT语句检索单个列。LIMIT 5指示MySQL返回不多于5行。

  • $\color{red}{注:}$
    • 行0：检索出来的第一行为行0而不是行1.因此，LIMIT 1，1将检索出第二行而不是第一行。
    • 在行数不够时：LIMIT中指定要检索的行数为检索的最大行数。如果没有足够的行，MySQL将只返回它能返回的那么多行。

条件查询

语法：SELECT 查询列表 FROM 表名 $\color{red}{where}$ 筛选条件;
解释：当where后的筛选条件成立为True，则表名可以筛选出来。若不成立为False，则该说明该数据不需要。

  1. $\color{blue}{过滤方式}$

    • 按条件表达式筛选
      • 条件运算符： > < =(等于) !=(不等于) <>(不等于) BETWEEN(在指定的两个值之间)
    • 按逻辑表达式筛选
      • 逻辑运算符：&&(与)[and] ||(或)[or] !(非)[not]
    • 模糊查询
      • like、between and、in、is null

  2. $\color{blue}{按条件表达式筛选}$

    比如：想查询年龄大于10的person信息；

    SELECT 
     *
FROM
     people
WHERE
     age > 10;

    比如：想查询部门编号不等于90的员工名和部门编号

    SELECT
     name,
     department_id
FROM
     employees
WHERE
     department_id <> 90; #或 != 90

  3. $\color{blue}{按逻辑表达式筛选}$

    比如：想去查询年龄在10到20之间的人的姓名；

    SELECT
     name
FROM
     people
WHERE
     age >= 10
AND
     age <= 20;

用通配符进行过滤

  1. $\color{blue}{通配符}$

    用来匹配值的一部分的特殊字符。

  2. $\color{blue}{搜索模式}$

    由字面值、通配符或两者组合成的搜索条件。

注：

为在搜索子句中使用通配符，必须使用LIKE操作符。LIKE指示Mysql，后跟的搜索模式利用通配符匹配而不是直接相等匹配进行比较。

百分号（%）通配符

最常使用的通配符是百分号（%）。在搜索串中，%表示任何字符出现任意次数。比如想找出所有以词John起头的人。

SELECT
        name
FROM
        person
WHERE
        name LIKE 'John%';

通配符可在搜索模式中任意位置使用，并且可以使用多个通配符

SELECT 
        prod_id, prod_name
FROM
        products
WHERE
        prod_name LIKE '%anvil%';

搜索模式’%anvil’表示匹配任何位置包含文本anvil的值，不论它之前或之后出现什么字符；
通配符也可以出现在搜索模式的中间，比如：找出所有$\color{red}{以s起头e结尾}$的产品。

SELECT
        prod_name
FROM
        products
WHERE
        prod_name LIKE 's%e';

注：

尾空格可能会干扰通配符匹配！！！

排序查询

语法：SELECT 查询列表 FROM 表 WHERE 条件 ORDER BY 排序列表 asc|desc;

比如：查询员工信息，要求工资从高到低排序；

SELECT
        *
FROM
        employees
ORDER BY
        salary DESC;#desc降序 esd升序(可省略)

  1. $\color{blue}{排序中加入筛选}$

    比如：想要查询部门编号大于等于90的员工信息，按入职时间先后排序；

    SELECT
     *
FROM
     employees
WHERE
     department_id >= 90
ORDER BY
     hiredate ASC;

  2. $\color{blue}{按多个列排序}$

    比如：想去检索产品id，产品价格和产品名字，并且按产品价格和产品名字排序；

    SELECT
     prod_id, prod_price, prod_name
FROM
     products
ORDER BY
     prod_price, prod_name;

$\color{purple}{注：}$

在给出ORDER BY子句时，应该保证它位于FROM子句之后。如果使用LIMIT，它必须位于ORDER BY之后，使用子句的次序不对将产生错误信息。一般ORDER BY子句都放在$\color{purple}{最后}$除了$\color{purple}{limit子句}$。

创建计算字段

Concat函数

作用：拼接串，把多个串连接起来形成一个较长的串。
比如：想要查询人民和国家，并且按 人名（国家）形式输出；

SELECT

文本处理函数

length

作用：获取参数值的字节个数；
mysql> SELECT LEGHTH(‘temp’);
mysql> SELECT LENGTH(‘你好’); #utf8下中文占三个字节

upper

作用：大小写转换，upper, lower
mysql> SELECT UPPER(‘hello’);
mysql> SELECT LOWER(‘HELLO’);

比如：把姓变大写，名小写，然后拼接

SELECT
        CONCAT(UPPER(last_name), LOWER(first_name)) AS 姓名
FROM 
        employees;

substr

作用：获取子串(substring)

  1. $\color{blue}{substr(string, pos)}$

    截取从下标索引为pos开始到字符串维的子串。$\color{red}{Mysql中的字符串下标索引从1开始}$

  2. $\color{blue}{substr(string, pos, len)}$

    截取从下标索引为pos开始的len个字符。（len代表字符长度，不是字节个数）

instr

instr(str, substr)：返回substr在str里的起始索引，如果找不到返回0。
mysql> SELECT INSTR(‘hello’, ‘e’);

trim

trim(str)：去除前后空格。
mysql> SELECT TRIM(‘ hello ‘);

trim(str1 FROM str2)：去除str2前后的str1.
mysql> SELECT TRIM(‘l’ FROM ‘hello’);

lpad

lpad(str, len, padstr)：在str的左边填充padstr直到总字符长度为len为止。如果len小于str原本的长度，则str缩短至len。同样的有rpad。
mysql> SELECT LPAD(‘Hi’, 10, ‘*’);

replace

replace(str, from_str, to_str)：把str中的from_str换成to_str。
mysql> SELECT REPLACE(‘HELLO’, ‘LLO’, ‘llo’);

日期和时间处理函数

日期和时间采用相应的数据类型和特殊的格式存储，以便能快速和有效的排序或过滤，别且节省物理空间。

now

  1. $\color{blue}{now()}$

    作用：返回当前系统日期+时间。
    mysql> SELECT NOW();

curdate

  1. $\color{blue}{curdate()}$

    作用：返回当前系统日期（不包含时间）。
    mysql> SELECT CURDATE();

curtime

  1. $\color{blue}{curtime()}$

    作用：返回当前系统时间（不包含日期）。
    mysql> SELECT CURTIME();

year

  1. $\color{blue}{year(Date)}$

    作用：获取指定日期的年份。
    mysql> SELECT YEAR(NOW());
    相同的还有Month, Day, Hour, Minute, Second, Time

str_to_date

  1. $\color{blue}{str_to_date()}$

    作用：将日期格式的字符转换成指定格式的日期
    mysql> SELECT STR_TO_DATE(‘1-1-2000’, ‘%m-%d-%y);

date_format

  1. $\color{blue}{date_format()}$

    作用：将日期转换成字符
    mysql> SELECT DATE_FORMAT(‘2000/1/1, ‘%Y年%m月%d日’);

时间戳操作

  1. 秒级别的时间戳 unix_timestamp(Date)

    作用：从1970-01-01 00：00：00到date按秒算。
    mysql> SELECT UNIX_TIMESTAMP(NOW());

  2. 当前时间 current_timestamp()

    作用：显示当前时间
    mysql> SELECT CURRENT_TIMESTAMP();

  3. FROM_UNIXTIME格式化时间戳函数 FROM_UNIXTIME()

    作用：将MySQL中以INT(11)存储的时间以”YYYY-MM-DD”格式来显示。
    语法：FROM_UNIXTIME(unix_timestamp,format);

附：format格式化字符

$\color{red}{注意事项}$

  1. 应该总是使用4位数字的年份。
  2. 如果要的是日期，请使用Date()
  3. MySQL 4.1.1中增加了许多日期函数，如果想要知道哪些函数可用，请自行查询。

数值处理函数

数值处理函数仅处理数值数据。这些函数主要运用于代数、三角或几何运算。

round

  1. $\color{blue}{round(X)}$

    作用：四舍五入。
    mysql> SELECT ROUND(1.4);

  2. $\color{blue}{round(X, D)}$

    作用：指定精度舍弃位数。
    mysql> SELECT ROUND(1.567, 2); #1.57

ceil

  1. $\color{blue}{ceil(X)}$

    作用：向上取整。返回大于等于该参数的最小整数。
    mysql> SELECT CEIL(1.5); #2

floor

  1. $\color{blue}{floor(X)}$

    作用：向下取整。返回小于等于该参数的最大整数。
    mysql> SELECT FLOOR(1.5); #2

truncate

  1. $\color{blue}{truncate(X, D)}$

    作用：截断。小数点后保留D位。
    mysql> SELECT TRUNCATE(1.65, 1); #1.6

mod

  1. $\color{blue}{mod(X, D)}$

    作用：取余。
    mysql> SELECT MOD(10, 3); #1

流程控制函数

  1. $\color{blue}{if}$

    if(exp1, exp2, exp3);
    作用：如果表达式1的值为true，则返回表达式2，否则返回表达式3.类似三目运算符。

  2. $\color{blue}{第一类case}$

    语法：

    case 字段或变量或表达式 
when 常量1 then 输出 
else 语句 
end;
类似switch case语句。

    比如：

    SELECT
     salary, department_id,
CASE
     department_id
WHEN 30 THEN salary * 1.1
WHEN 40 THEN salary * 1.2
WHEN 50 THEN salary * 1.3
ELSE salary
END

  3. $\color{blue}{第二类case}$

    语法：

    case
when 条件1 then 输出1
when 条件2 then 输出2
else 输出3
end
类似多重if

分组函数

简介:用作统计使用，又称聚合函数、统计函数或组函数。

聚集函数

运行在行组上，计算和返回单个值的函数。

  1. $\color{blue}{AVG}$

    返回某列的平均值。
    $\color{red}{注}$：AVG()只能用来确定特定数值列的平均值，而且列名必须作为函数参数给出，为了获得多个列的平均值，必须使用多个AVG()函数。
    $\color{red}{NULL值}$：AVG()函数忽略列值为NULL的行

  2. $\color{blue}{COUNT}$

    返回某列的行数。
    $\color{red}{NULL值}$：如果指定列名，则指定列的值为空的行被COUNT()函数忽略，但如果COUNT()函数中用的是*号，则不忽略。

  3. $\color{blue}{MAX}$

    返回某列的最大值。
    $\color{red}{对非数值数据使用MAX()}$：虽然MAX()一般用来找出最大的数值或日期值，但MySQL允许将它用来返回任意列中的最大值，包括返回文本列中的最大值。在用于文本数据时，如果数据按相应的列排序，则MAX()返回最后一行。

  4. $\color{blue}{MIN}$

    返回某列的最小值。

  5. $\color{blue}{SUM}$

    返回某列值之和。

使用

比如：想要去查询员工工资的总和：

SELECT SUM(salary) FROM employees;
#其余类似不做举例

聚集不同值

上述5个聚集函数都可以如下进行使用：

对所有的行执行计算，指定ALL参数或不给参数（因为ALL是默认行为）。
只包含不同的值，指定DISTINCT参数。

DISTINCT

利用DISTINCT可以做到去重效果。

SELECT SUM(DISTINCT salary) FROM employees;

$\color{red}{注：}$

如果指定列名，则DISTINCT只能用于COUNT()、不能用于COUNT(*)，因此不能使用COUNT(DISTINCT)。DISTINCT必须使用列名，不能用于计算或者表达式。

COUNT函数

  1. $\color{blue}{count(x)}$

    作用：返回某列的行数。
    count(*)统计所有字段。
    count(1)统计1的个数，加一列全是1的列。（并不是只能是1）

  2. $\color{blue}{效率}$

    MYISAM存储引擎下，count(*)的效率最高。
    原因：在引擎内部有一个计数器，直接返回个数。
    INNODB存储引擎下，count(*)和count(1)效率近似快，但是比count(x)快。
    原因：count(x)需要做判断，进行筛选。
