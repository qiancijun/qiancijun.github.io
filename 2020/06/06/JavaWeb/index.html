Web相关

  1. 软件架构

    • C/S：客户端/服务器
    • B/S：浏览器/服务器

  2. 资源分类

    • 静态资源：用户访问后得到的结果都是一样的，称为静态资源，静态资源可以直接被浏览器解析
    • 动态资源：每个用户访问后得到的结果可能不一样。动态资源被访问后，先要转换为静态资源，再返回给浏览器

  3. 网络通信三要素

    • IP
    • 端口
    • 传输协议

Tomcat

  • 服务器： 安装了服务器软件的计算机
  • 服务器软件：接受用户的请求，处理请求，做出响应

  • web服务器软件：接受用户的请求，处理请求，做出响应。

    • 在web服务器软件中，可以部署web项目，让用户通过浏览器来访问这些项目
    • web容器：动态资源必须在容器中才能运行

  • 常见的web服务器软件：

    1. webLogic：Oracle，大型的JavaEE服务器，支持所有的JavaEE规范
    2. webSphere：IBM，大型的JavaEE服务器，支持所有的JavaEE规范
    3. JBOSS：JBOSS，大型的JavaEE服务器，支持所有的JavaEE规范
    4. Tomcat：Apache基金组织，中小心JavaEE服务器，仅支持少量的JavaEE规范servlet/jsp，开源免费
      • 一般会将tomcat的默认端口号修改为80.80端口是http协议的默认端口号。好处：在访问时不用输入端口号

生命周期

  1. init

    • 初始化方法
    • 在servlet被创建时执行，只会执行一次

  2. service

    • 提供服务的方法
    • 每一次servlet被访问时执行。执行多次

  3. destroy

    • 销毁方法
    • 在服务器正常关闭时执行

  4. getServletConfig

    • 获取ServletConfig对象

  5. getServletInfo

    • 获取servlet的信息

  • servlet的生命周期

    1. 被创建：执行init方法，只执行一次
      • 默认情况下，第一次被访问时，Servlet被创建
      • 可以配置Servlet创建时机，在<servlet>标签下配置
        • 第一次被访问时创建：<load-on-startup>的值为负数，默认为负数
        • 在服务器启动时创建：<load-on-startup>的值为0或正整数<load-on-startup>5</load-on-startup>
    2. 提供服务：执行service方法，会执行多次
      • 每次访问servlet时，service方法都会被调用一次
    3. 被销毁：执行destory方法，只执行一次
      • servlet被销毁时执行，服务器关闭时，servlet被执行
      • 只有服务器正常关闭时才会执行destory方法
      • 在servlet被销毁之前执行，一般用来释放资源

  • servlet的init方法，只执行一次，说明一个servlet在内存中只存在一个对象，Servlet是单例的。多个用户同时访问时，可能存在线程安全问题。

    • 解决方法：尽量不要在servlet中定义成员变量，即使定义了成员变量也不要对其修改

HTTP协议

  • Hyper Text Transfer Protocol：超文本传输协议

    • 传输协议：定义了客户端和服务器端通信时，发送数据的格式
    • 特点：
      1. 基于TCP/IP的高级协议
      2. 默认端口号：80
      3. 基于请求/响应模型，一次请求对应一次响应
      4. 无状态：每次请求之间相互独立，不能交互数据

  • 请求数据：客户端发送给服务器端的数据

  • 响应数据：服务器端发送给客户端的数据

请求消息数据格式

  1. 请求行
    • 请求方式 请求url 请求协议/版本
    • GET /index.html HTTP/1.1
  2. 请求头：浏览器告诉服务器一些信息
    • 请求头名称：请求头值
    • 常见的请求头：
      • User-agent：浏览器告诉服务器，浏览器版本信息。可以在服务器端获取该信息，解决浏览器的兼容性问题
      • Referer：告诉服务器，当前请求从哪里来。可以防止被盗取链接，做统计工作
      • Connection：keep-alive 连接可以被复用
  3. 请求空行
    • 空行，用于分割POST请求头和请求体
  4. 请求体（正文）
    • 封装POST请求消息的请求参数
  • 请求方式：HTTP有7中请求方式，常用的有2种
    • GET：
      1. 请求参数在请求行中，在url后
      2. 请求的url长度有限制
      3. 不太安全
    • POST
      1. 请求参数在请求体中
      2. 请求的url长度无限制
      3. 相对安全

响应消息数据格式

  1. 响应行
    • 组成：协议/版本 响应状态码 状态码描述
    • 响应状态码：服务器告诉客户端浏览器本次请求和响应的状态，状态码都是三位数字，有五种分类：
      • 1XX：服务器接受客户端消息，但没有接受完成，等待一段时间后发送1XX状态码
      • 2XX：成功。
      • 3XX：302重定向，304访问缓存
      • 4XX：失败，客户端错误。404路径没有对应的资源，405请求方式没有对应的方法
      • 5XX：失败，服务器端错误。500服务器内部出现异常
  2. 响应头：
    • 格式：头名称：值
    • 常见的响应头：
      • Content-type：服务器告诉客户端本次响应体数据格式以及编码格式
      • Content-dispostion：服务器告诉客户端以什么格式打开响应体数据，默认值in-line（在当前页面中打开），可选值attachment;filename=xxx（以附件形式打开响应体，文件下载）
  3. 响应空行
  4. 响应体：传输的数据

Request

  • 步骤：
    1. tomcat服务器会根据请求url中的资源路径，创建对应的ServletDemo1的对象
    2. tomcat服务器会创建request和response对象，request对象中封装请求消息数据
    3. tomcat将request和response两个对象传递给sevice方法，并且调用sevice方法
    4. Java程序可以通过request对象获取请求消息数据，通过response对象设置响应消息数据
    5. 服务器在给浏览器做出响应之前，会从response对象中拿出Java程序返回的响应消息数据
  • request对象和response对象原理：
    1. request和response对象是由服务器创建的。
    2. request对象是来获取请求消息的，response对象是来设置响应消息

request功能

  • 获取请求消息数据

会话技术

  • 会话：一次会话中包含了多次请求和响应。
    • 一次会话：浏览器第一次给服务器资源发送请求，会话建立，直到有一方断开为止
    • 功能：在一次会话的范围内多次请求间，共享数据
    • 方式：
      1. 客户端会话技术：Cookie
      2. 服务器端会话技术：Session

Cookie

  • 概念：客户端会话技术，将数据保存到客户端
  • 使用步骤：
    1. 创建Cookie对象，指定数据
      • Cookie(String name, String value)
    2. 发送Cookie对象
      • response.addCookie(Cookie cookie)
    3. 获取Cookie，拿到数据
      • Cookie[] request.getCookies()
  • 实现原理：基于响应头set-cookie和请求头cookie实现
  • 细节：
    1. 一次可以发送多个cookie
    2. 当浏览器关闭后，Cookie数据被销毁
    3. 持久化存储： setMaxAge(int seconds)
      • 正数：将Cookie数据写到硬盘的文件中。参数为cookie存活时间
      • 负数：默认值
      • 零： 删除cookie信息
    4. 在tomcat8之前，cookie不能直接存储中文， 在8之后支持中文数据
      • 8之前需要将中文数据转码，一般采用URL编码(%两个十六进制数)
    5. 多个项目的cookie是不能共享的
      • setPath(string path)：设置cookie的获取范围。默认情况下会去设置当前项目的虚拟目录
      • 如果要共享，可以将path设置为/
      • 不同的tomcat服务期间数据共享：
        • setDomain(String path)：如果设置一级域名相同，那么多个服务器之间cookie可以共享

  • 特点：

    1. cookie存储数据在客户端浏览器，不安全
    2. 浏览器对于单个cookie有限制（4KB），以及对同一个域名下的总cookie数量也有限制（20以内）

  • 作用：

    1. cookie一般用于少量的不太敏感的数据
    2. 在不登陆的情况下，完成服务器对客户端的身份识别

JSP

  • 概念：Java Server Pages java服务器端页面，一个特殊的页面，其中既可以直接定义html标签，也可以定义java代码
  • 原理：
    1. jsp本质上就是一个servlet
  • JSP的脚本：JSP定义Java代码的方式
    1. <% Java代码 %>：定义的Java代码，在service方法中。service方法中可以定义什么，该脚本中就可以定义什么
    2. <%! Java代码 %>：定义的Java代码在jsp转换后的java类的成员位置
    3. <%= Java代码 %>：定义的Java代码会输出到页面上，输出语句中可以定义什么，该脚本就可以定义什么
  • JSP的内置对象：在JSP页面中不需要获取和创建的对象，可以直接使用
    • 一共有9个内置对象
      • request
      • response
      • out：字符输出流对象
  • 原理：Session的实现是依赖于Cookie的
  • 细节：
    1. 当客户端关闭，服务器不关闭，两次获取的Session不是同一个，如果需要相同，可以设置Cookie来保存
    2. 当客户端不关闭，服务器关闭，两次获取的Session不是同一个，但是要确保数据不丢失
      • Session的钝化：在服务器正常关闭之前，将Session对象序列化到硬盘上
      • Session的活化：在服务器启动后，将Session文件反序列化
    3. Session销毁
      • 服务器关闭
      • Session对象调用invalidate()
      • 默认三十分钟失效

MVC开发模式

  1. jsp演变历史
    • 早期只有servlet，只能使用response输出标签数据，非常麻烦
    • sun公司用jsp来简化了servlet的开发，如果过渡使用jsp，难于维护，难于分工合作
    • javaweb开发借鉴mvc开发模式，使得程序的设计更加合理
  2. MVC：
    • M：model，模型，JavaBean，完成具体的业务操作，如查询数据库，封装对象
    • V：Viem，视图，Jsp，展示数据
    • C：Controller，控制器，Servlet，获取用户输入，调用模型，将数据交给视图进行展示
  3. 优点：
    • 耦合性低，方便维护，可以利于分工
    • 重用性高
  4. 缺点：
    • 使得项目架构变得复杂，对开发人员要求高

EL表达式

  • 概念：Expression Language 表达式语言
  • 作用：替换和简化jsp页面中Java代码的编写
  • 语法：前加\
  • 使用：
    1. 运算
      • 算术运算符： + - * /(div) %(mod)
      • 比较运算符： > < >= <= == !=
      • 逻辑运算符： &&(and) ||(or) !(not)
      • 空运算符： empty 用于判断字符串、集合和数组长度是否为0
    2. 获取值
      • EL表达式只能从域对象中获取值
      • 语法：${域名称.键名称}，从指定域中获取指定键的值
      • 域名称：
        1. pageScope -> pageContext
        2. requestScope -> request
        3. sessionScope -> session
        4. applicationScope -> application(ServletContext)
      • 例：在request中存储了name=qiancijun
      • 获取：${requestScope.name}
      • 省略域名称：${键名称}，表示依次从最小的域中查找是否有该键对应的值，直到找到为止
    3. 获取对象的值
      • 通过对象的属性来获取，setter和getter方法，去掉get和set，在将剩余部分首字母变小写
    4. 获取List集合的值
      • ${域名称.键名[索引]}
    5. 获取map集合的值
      • ${域名称.键名.key名称}
      • ${域名称.键名["key名称"]}
  • 隐式对象：el表达式中有11个隐式对象，不用创建可以直接使用
    • pageContext：获取jsp其它8个对象

JSTL

  • 概念：JavaServer Pages Tag Library JSP标准标签库
  • 作用：用于简化和替换页面上的Java代码
  • 步骤：
    1. 导入jar包
    2. 引入标签库：taglib指令：<%% taglib>
    3. 使用标签
  • 常用的JSTL标签
    1. if：相当于Java中的if
      • test：必须属性，接受boolean表达式，如果表达式为true则显示if标签体内容，如果为false则不显示标签体内容，一般情况下会结合el表达式一起使用
    2. choose：相当于Java中的switch
      • 使用choose标签声明
      • 使用when标签做判断
      • 使用otherwise标签做其它情况的声明
    3. foreach：相当于Java中的for
      • 属性：
        • begin 开始值
        • end 结束值
        • var 临时变量
        • step 步长
        • varStatus 循环状态
          • index：容器中元素的索引，从0开始
          • count：循环的次数

Filter

  • 当访问服务器资源式，过滤器可以将请求拦截下来，完成一些特殊的功能
  • 作用：一般完成一些通用的操作。如：登录验证，统一编码处理
  • 步骤：
    1. 定义一个类，实现Filter接口
    2. 复写方法
    3. 配置拦截路径
  • 细节：
    1. 过滤器生命周期：
      • init：在服务器启动后，会创建Filter对象，然后调用init方法。只执行一次，用于加载资源
      • doFilter：每一次请求被拦截资源时，会执行，执行多次
      • destory：在服务器关闭后，Filter对象被销毁。如果服务器是正常关闭，则会执行destory方法。只执行一次，用于释放资源
    2. 过滤器配置：
      • 拦截路径配置：
        1. 具体资源路径： /index.jsp
        2. 目录拦截：/user/* 访问user下的所有资源时，过滤器都会被执行
        3. 后缀拦截：*.jsp 访问所有jsp资源时，过滤器都会被执行
        4. 拦截所有：/*
      • 拦截方式配置：资源被访问的方式
        • 注解配置
          • dispatcherTypes属性：可以配置多个值，用大括号包裹
            1. REQUEST：默认值。浏览器直接请求资源
            2. FORWARD：转发访问资源
            3. INCLUDE：包含访问资源
            4. ERROR：错误跳转资源
            5. ASYNC：异步访问资源
        • web.xml配置
      1. 过滤器链
        • 执行顺序：如果有两个过滤器，分别是1和2。过滤器1先执行，过滤器2后执行，过滤器2先返回，过滤器1后返回
        • 先后顺序：按照类名的字符串比较顺序，值小的先执行

Listener

  • 事件监听机制：
    1. 事件
    2. 事件源：事件发生的地方
    3. 监听器：一个对象或一段代码
    4. 注册监听：将事件、事件源和监听器绑定在一起，当事件源上发生某个事件后执行监听器的代码
  • ServletContextListener：监听ServletContext对象的创建和销毁
  • 步骤：
    1. 定义一个类，实现接口
    2. 复写方法
    3. 配置
