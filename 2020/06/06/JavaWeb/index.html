Web相关

  1. 软件架构

    • C/S：客户端/服务器
    • B/S：浏览器/服务器

  2. 资源分类

    • 静态资源：用户访问后得到的结果都是一样的，称为静态资源，静态资源可以直接被浏览器解析
    • 动态资源：每个用户访问后得到的结果可能不一样。动态资源被访问后，先要转换为静态资源，再返回给浏览器

  3. 网络通信三要素

    • IP
    • 端口
    • 传输协议

Tomcat

  • 服务器： 安装了服务器软件的计算机

  • 服务器软件：接受用户的请求，处理请求，做出响应

  • web服务器软件：接受用户的请求，处理请求，做出响应。

    • 在web服务器软件中，可以部署web项目，让用户通过浏览器来访问这些项目
    • web容器：动态资源必须在容器中才能运行

  • 常见的web服务器软件：

    1. webLogic：Oracle，大型的JavaEE服务器，支持所有的JavaEE规范
    2. webSphere：IBM，大型的JavaEE服务器，支持所有的JavaEE规范
    3. JBOSS：JBOSS，大型的JavaEE服务器，支持所有的JavaEE规范
    4. Tomcat：Apache基金组织，中小心JavaEE服务器，仅支持少量的JavaEE规范servlet/jsp，开源免费
      • 一般会将tomcat的默认端口号修改为80.80端口是http协议的默认端口号。好处：在访问时不用输入端口号

部署

  1. 直接将项目放到webapps目录下即可
    localhost:8080/文件夹名称/文件名
    简化部署：将项目打包成war包
  2. 配置conf/serve.xml文件
    在host标签下添加
    <Context docBase="项目路径" path="虚拟路径"/>
    这种方式不安全，会使其它的项目无法运行
  3. 在conf/Catalina文件夹下创建一个xml文件
    <Context docBase="项目路径"，虚拟目录就为xml文件的名称

目录结构

  • java动态项目的目录结构：
    • –项目的根目录
      • –WEB-INF目录
        • web.xml：web项目的核心配置文件
        • classes目录：字节码文件目录
        • lib目录：依赖的jar包

Servlet

  • Servlet：Server applet，运行在服务器端的小程序。
    • 本质上就是一个接口，定义了Java类被浏览器访问到（tomcat识别）的规则。
    • 自定义一个类，实现servlet接口，复写方法

实现步骤

  1. 创建一个JavaEE项目
  2. 定义一个类，实现servlet接口
  3. 实现接口中的抽象方法
  4. 配置servlet
    在web.xml中配置，注意url中有/
  • 原理：
    1. 浏览器输入地址，首先访问到localhost主机地址，并且通过8080访问到tomcat应用程序
    2. 通过虚拟路径找到要访问的项目名
    3. 通过/demo01，在web.xml中找到对应的全类名
    4. 找到全类名之后，通过反射，tomcat将全类名对应的字节码文件加载进内存。Class.forName()
    5. 创建对象class.newInstance();
    6. 调用service方法

生命周期

  1. init

    • 初始化方法
    • 在servlet被创建时执行，只会执行一次

  2. service

    • 提供服务的方法
    • 每一次servlet被访问时执行。执行多次

  3. destroy

    • 销毁方法
    • 在服务器正常关闭时执行

  4. getServletConfig

    • 获取ServletConfig对象

  5. getServletInfo

    • 获取servlet的信息

  • servlet的生命周期

    1. 被创建：执行init方法，只执行一次
      • 默认情况下，第一次被访问时，Servlet被创建
      • 可以配置Servlet创建时机，在<servlet>标签下配置
        • 第一次被访问时创建：<load-on-startup>的值为负数，默认为负数
        • 在服务器启动时创建：<load-on-startup>的值为0或正整数<load-on-startup>5</load-on-startup>
    2. 提供服务：执行service方法，会执行多次
      • 每次访问servlet时，service方法都会被调用一次
    3. 被销毁：执行destory方法，只执行一次
      • servlet被销毁时执行，服务器关闭时，servlet被执行
      • 只有服务器正常关闭时才会执行destory方法
      • 在servlet被销毁之前执行，一般用来释放资源

  • servlet的init方法，只执行一次，说明一个servlet在内存中只存在一个对象，Servlet是单例的。多个用户同时访问时，可能存在线程安全问题。

    • 解决方法：尽量不要在servlet中定义成员变量，即使定义了成员变量也不要对其修改

Servlet3.0

  • 优点：支持注解配置。可以不需要web.xml
  • 步骤：
    1. 创建JavaEE项目，选择Servlet版本为3.0以上，可以不创建web.xml
    2. 定义类，实现servlet接口
    3. 复写方法
    4. 在类上使用注解@webServlet进行配置
      • @WebServlet(“/资源路径”)

体系结构

  • GenericServlet：实现了Servlet接口，是一个抽象类
    • 将Servlet接口中的其它的方法做了默认的空实现，只将service方法作为抽象
    • 定义Servlet类时，可以继承GenericServlet，实现service方法
  • HttpServlet：GenericServlet的子类，也是一个抽象类。
    • 对HTTP协议的一种封装，简化操作
    • 定义类继承HttpServlet
    • 复写doGet或doPOST方法

相关配置

  1. urlpartten：Servlet访问路径
    • 一个Servlet可以定义多个访问路径@WebServlet({"/d1", "/demo1", "/demo1"})
  2. 路径定义规则
    • /xxx
    • /xxx/xxx：多层路径，目录结构
    • *.xxx

HTTP协议

  • Hyper Text Transfer Protocol：超文本传输协议
    • 传输协议：定义了客户端和服务器端通信时，发送数据的格式
    • 特点：
      1. 基于TCP/IP的高级协议
      2. 默认端口号：80
      3. 基于请求/响应模型，一次请求对应一次响应
      4. 无状态：每次请求之间相互独立，不能交互数据

请求消息数据格式

  1. 请求行
    • 请求方式 请求url 请求协议/版本
    • GET /index.html HTTP/1.1
  2. 请求头：浏览器告诉服务器一些信息
    • 请求头名称：请求头值
    • 常见的请求头：
      • User-agent：浏览器告诉服务器，浏览器版本信息。可以在服务器端获取该信息，解决浏览器的兼容性问题
      • Referer：告诉服务器，当前请求从哪里来。可以防止被盗取链接，做统计工作
      • Connection：keep-alive 连接可以被复用
  3. 请求空行
    • 空行，用于分割POST请求头和请求体
  4. 请求体（正文）
    • 封装POST请求消息的请求参数
  • 请求方式：HTTP有7中请求方式，常用的有2种
    • GET：
      1. 请求参数在请求行中，在url后
      2. 请求的url长度有限制
      3. 不太安全
    • POST
      1. 请求参数在请求体中
      2. 请求的url长度无限制
      3. 相对安全

响应消息数据格式

Request

  • 步骤：
    1. tomcat服务器会根据请求url中的资源路径，创建对应的ServletDemo1的对象
    2. tomcat服务器会创建request和response对象，request对象中封装请求消息数据
    3. tomcat将request和response两个对象传递给sevice方法，并且调用sevice方法
    4. Java程序可以通过request对象获取请求消息数据，通过response对象设置响应消息数据
    5. 服务器在给浏览器做出响应之前，会从response对象中拿出Java程序返回的响应消息数据
  • request对象和response对象原理：
    1. request和response对象是由服务器创建的。
    2. request对象是来获取请求消息的，response对象是来设置响应消息

体系结构

  • ServletRequest：接口

  • HttpServletRequest：接口，继承了ServletRequest

  • org.apache.catalina.connector.RequestFacade：类（tomcat）实现了HttpServletRequest接口
