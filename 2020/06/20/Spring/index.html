Spring框架概述

  1. Spring是一个轻量级的开源的JavaEE框架
  2. 解决企业应用开发的复杂性
  3. Spring有两大核心部分：IOC和AOP
    • IOC：控制反转，把创建对象的过程交给Spring管理
    • AOP：面向切面，不修改源代码的情况下进行功能增强
  4. 特点：
    • 方便解耦，简化开发
    • AOP编程、声明式事务支持
    • 方便测试
    • 方便继承各种优秀框架

IOC容器

  • IOC：控制反转，是面向对象编程中的一种设计原则，可以用来减低计算机代码之间的耦合度。通过控制反转， 对象在被创建的时候，由一个调控系统内所有对象的外界实体将其所依赖的对象的引用传递给它。也可以说，依赖被注入到对象中

¶底层原理

  • 过程
    1. 通过配置文件解析xml，获得类的路径
    2. 通过工厂模式返回对象
    3. 通过反射，用解析出的路径，利用反射机制创建对象

¶BeanFactory

  • IOC思想基于IOC容器完成，IOC容器底层就是对象工厂
  • Spring提供IOC容器实现的两种方式：（两个接口）
    1. BeanFactory
      • IOC容器基本实现，是Spring内部的使用接口，一般不提供给开发人员使用
      • 加载配置文件时不会创建对象，在获取或者使用的时候才会创建
    2. ApplicationContext
      • BeanFactory接口的子接口，提供更多更强大的功能，一般面向开发人员使用
      • 加载配置文件时候就会把配置文件对象进行创建
      • 主要的实现类：
        • FileSystemXmlApplicationContext
        • ClassPathXmlApplicationContext

¶FactoryBean

  • Spring有两种类型Bean，一种普通bean，另外一种工程bean
    1. 普通bean：在配置文件中定义bean类型就是返回类型
    2. 工程bean：在配置文件定义bean类型返回类型可以不一样
  • 创建工厂bean步骤
    1. 创建类，让这个类作为工厂bean，实现接口FactoryBean
    2. 实现接口里面的方法，在实现的方法中定义返回bean的类型

¶Bean的作用域

  • 在Spring例可以设置创建bean实例是单实例还是多实例。默认情况下，创建的bean是一个单实例对象
  • 设置多实例：
    • 在spring配置文件中，bean标签中的scope属性设置，常用值
      1. singleton，表示是单实例对象
      2. prototype，表示是多实例对象
      3. request，域对象，一次请求
      4. session，域对象，一次会话
    • singleton与prototype区别：
      1. singleton -> 单实例；prototype -> 多实例
      2. 设置scope值是singleton时，加载spring配置文件时就会创建一个单实例对象
      3. 设置scope值是prototype时，不是在加载配置文件时创建对象，在调用getBean方法创建多实例对象

¶Bean生命周期

  • 生命周期：从对象创建到对象销毁的过程

  • Bean声明周期：

    1. 通过构造器创建bean实例（无参构造）
    2. 设置bean属性值和对其他bean的引用（调用set方法）
    3. 调用bean的初始化方法（需要配置）
    4. 完成创建bean
    5. 当容器关闭时，调用销毁方法（需要配置，并且手动调用）

  • Bean的后置处理器，声明周期一共有七步：在初始化之前和之后，把bean实例传递给bean后置处理器。执行postProcessBeforeInitialization和postProcessAfterInitialization方法

    • 创建类：实现接口BeanPostProcessor，创建后置处理器

AOP

  • AOP：Aspect Oriented Programming面向切面编程，通过预编译和运行期间动态代理实现程序功能的统一维护的一种技术，AOP是OOP的延续。利用AOP可以对业务逻辑的各个部分进行隔离，从而使得业务逻辑各部分之间的耦合度降到最低，提高程序的可重用性，同时提高了开发的效率
  • 主要目的：将日志记录，性能统计，安全控制，事务处理，异常处理等代码从业务逻辑代码中划分出来，通过对这些行为的分离，我们希望可以将它们独立到非指导业务逻辑的方法中，进而改变这些行为的时候不影响业务逻辑的代码
  • 通俗的说，不通过修改源代码的方式，在业务里添加新的功能

¶底层原理

  • 底层使用了动态代理来增强方法功能，有两种情况

    1. 有接口情况，使用JDK动态代理

    2. 没有接口情况，使用CGLIB动态代理
      • 创建当前类的子类的代理对象，增强类的方法

¶AOP术语

  1. 连接点
    • 类中的哪些方法可以被增强，那么这些方法就被称为连接点
  2. 切入点
    • 实际被真正增强的方法称为切入点
  3. 通知（增强）
    • 实际增强的逻辑部分称为通知
    • 通知有多种类型：
      1. 前置通知：@Befoe在方法前增强
      2. 后置通知：@AfterReturning在方法后增强，在返回结果之后执行
      3. 环绕通知：@Around前后都增强
      4. 异常通知：@AfterThrowing当发放出现异常时增强
      5. 最终通知：@After类似finnally，，方法有异常也会执行
  4. 切面
    • 把通知应用到切入点的过程

¶AOP操作

  • Spring框架一般基于AspectJ实现AOP操作
    • AspectJ：不是Spring的组成部分，它是一个独立的AOP框架，一般把AspectJ和Spring一起使用，进行AOP相关操作
  • 基于AspectJ实现AOP操作
    1. 基于XML配置文件实现
    2. 基于注解方式实现
  • 步骤
    1. 引入AOP相关的依赖jar包
    2. 切入点表达式：对指定类指定方法进行增强
      • 语法结构：execution([权限修饰符] [返回类型] [类全路径] ([参数列表]))
      • 例：对cn.qiancijun.spring.dao.UserDao类中的add()方法进行增强
      • exection(* cn.qiancijun.spring.dao.UserDao.add(..))：*表示任意修饰符，返回类型可以省略，参数列表用..代替
      • 例：对cn.qiancijun.spring.dao.UserDao类中的所有方法进行增强
      • exection(* cn.qiancijun.spring.dao.UserDao.*(..))
      • 例：对cn.qiancijun.spring.dao包中的所有类的所有方法进行增强
      • exection(* cn.qiancijun.spring.dao.*.*(..))

¶事务操作

  • 事务一般添加到JavaEE三层架构中的Service层上
  • Spring事务管理操作：一般使用第二种
    1. 编程式事务管理
    2. 声明式事务管理（注解或xml配置）
  • 在Spring进行声明式事务管理，底层使用了AOP。Spinrg提供了一个接口PlatformTransactionManager，代表事务管理器，这个接口针对不同的框架提供了不同的实现类

¶注解声明式事务管理

  1. 在Spring配置文件配置事务管理器
  1. 开启事务注解
    • 引入名称空间tx
  1. 在service类上（或service里方法上）添加事务注解@Transactional
    • 如果把这个注解添加到类上面，表示这个类里面所有的方法都添加事务
    • 把注解添加到方法上，表示只为这个方法添加事务
  • 参数配置：
    1. propagation：事务传播行为propagation = Propagation.REQUIRED
      • 多事务方法直接进行调用，这个过程中事务是如何进行管理的
      • 事务方法：对数据库表数据进行变化的操作
      • 一共又7种传播行为：
        1. REQUIRED:默认值。如果又事务在运行，当前的方法就在这个事务内运行，否则就启动一个新的事务，并在自己的事务内运行
        2. REQUIRED_NEW：当前的方法必须启动新事务，并在它自己的事务内运行，如果又事务正在运行，应该将它挂起
        3. SUPPORTS：如果有事务在运行，当前的方法就在这个事务内运行，否则它可以不运行在事务种
        4. NOT_SUPPORTED：当前的方法不应该运行在事务中，如果有运行的事务，将它挂起
        5. MANDATORY：当前的方法必须运行在事务内部，如果没有正在运行的事务，就抛出异常
        6. NEVER：当前的方法不应该运行在事务中，如果有运行的事务，就抛出异常
        7. NESTED：如果有事务在运行，当前的方法就应该在这个事务的嵌套事务内运行，否则，就启动一个新的事务，并在它自己的事务内运行
    2. isolation：事务的隔离级别
    3. timeout：超时时间
    4. readOnly：是否只读
    5. rollbackFor：回滚
    6. noRollbackFor：不回滚
