Gin

Gin 是一个用 Go (Golang) 编写的 web 框架。 它是一个类似于 martini 但拥有更好性能的 API 框架, 由于 httprouter，速度提高了近 40 倍。

特点

  1. 快速：基于 Radix 树的路由，小内存占用。没有反射。可预测的 API 性能。
  2. 支持中间件：传入的 HTTP 请求可以由一系列中间件和最终操作来处理。 例如：Logger，Authorization，GZIP，最终操作 DB。
  3. 崩溃处理：Gin 可以 catch 一个发生在 HTTP 请求中的 panic 并 recover 它。这样，你的服务器将始终可用。例如，你可以向 Sentry 报告这个 panic！
  4. JSON验证
  5. 路由组
  6. 错误管理

模型绑定与验证

将请求体与相对应的类型绑定，仅支持JSON、XML、YAML、标准表单格式。Gin提供了两种绑定模型的方法。

  • Must Bind：底层使用MustBindWith，如果发生绑定错误会触发c.AbortWithError(400, err).SetType(ErrorTypeBind)如果想控制错误返回，应该使用Should Bind
  • Should Bind：底层使用ShouldBindWith，如果发生错误，开发者应自行处理。
